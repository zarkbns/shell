The problem with wallet security today

The old way

  • Browser extensions that sit outside the wallet
  • Security warnings that appear after approval
  • Separate apps users have to remember to open
  • No awareness of what a transaction actually does

Shell

  • Lives inside the signing pipeline, not beside it
  • Intercepts before the user sees the confirmation
  • Always on — no setup per transaction
  • Reads instruction data to know what the tx actually does

How Shell works

Pipeline Intercept

Plugs into the wallet's pre-flight handler. Every transaction is checked before the user sees a confirmation screen.

Away Mode

Sets a local flag that blocks all background signing requests the moment the user steps away.

Mimic Detection

Catches addresses engineered to look like known contacts using heuristic and visual normalization checks.

On-chain Verify

Queries Solana mainnet RPC directly to flag zero-history accounts and program destinations.

Integration guide

Install

Terminal 
npm install @shellxyz/sdk

Initialize inside your wallet client

setup.ts 
import ShellSDK from '@shellxyz/sdk'

ShellSDK.init({
  contactList: walletClient.getSavedAddresses(),
  solThreshold: 10
})

Intercept transactions before signing

interceptor.ts 
async function onTransactionRequest(tx) {
  const result = await ShellSDK.analyzeTransaction({
    address: tx.destination,
    transferAmount: tx.amount,
    transactionType: tx.type
  })

  if (result.blocked) return wallet.reject(result.reason)
  if (result.requiresPIN) return wallet.triggerPIN()
  return wallet.proceedToSign(tx)
}

Deep decode — read what the transaction actually does

decoder.ts 
const decoded = await ShellSDK.decodeAndAnalyze({
  serializedTransaction: tx.serialized,
  encoding: 'base64',
  transferAmount: tx.amount,
  contactList: wallet.contacts
})

if (decoded.containsUnknownPrograms) {
  wallet.showWarning(decoded.warning)
}

Away Mode — block background signing

awayMode.ts 
// User locks their screen or goes idle
ShellSDK.setAwayMode(true)

// Any background signing attempt is blocked instantly
// Re-enable when user returns
ShellSDK.setAwayMode(false)
